Intype - 像Textmate一样好用的文本编辑器

纯文本编辑是开发人员必不可少的工具（你不要跟我讲你用Eclipse或Visula Studio）。

记得上大学的时候纯文本编辑一直都是用Windows自带的 记事本 - Notepad，以前的Java教师也曾经推荐我用Editplus，但因为它哪难看的UI设计的界面布局，我一直没有用它。

后面出来工作以后，在某天发现了 EmEditor 这个软件，才发现之间一直用Notepad的愚蠢。所谓“工欲善其事，必先利其器”，有一个好用的工具，事情会做得更上手。的确是这样，用了 EmEditor 以后很多以前复杂的事情都变简单的很多，比如查找替换。

但因为 EmEditor 是收费软件，后面我也一直在寻找一个比较更好的。

去年的时候发现了Notepad++，这个是SourceForge最火的文本编辑器，功能也的确很不错，很多都比 EmEditor 强了，但惟有一点不爽的就是它的正则查找替换一直都用不习惯，不知道它加了些什么东西。
后面一段时间我基本上是在用 Notepad++ 了，但Notepad++的界面布局我也不是很喜欢，感觉看起来很臃肿，而且带了一大堆我从来不会用的插件。

可以后面出现 西藏事件 以后，Notepad++发起了 抵制北京奥运 的事件，再接连上引起了 SourceFore被封事件 哎，没SourceFore这日子怎么过呀！

从此我决定再也不用Notepad++了！

正好这个时候发现了一个模仿 Mac OSx 里面的 Textmate 这个软件的工具：E - TextEditor，这个就比较适合我了，它自带了和 Textmate 一样的代码高亮风格。不过还是不太喜欢，一个是它对中文支持不太好，另外不喜欢它的名字和UI。

我也曾经试过 Vim,Emacs 一直都听很多人说是神器，但太复杂了，搞着累，我就想简简单单的看看文件，小修改一下。

今天我发现了 Intype 这个软件，它和E - TextEditor基本相似，也是模仿 Textmate 的，而且我最喜欢的就是它的 UI 设计，很细腻，布局排版都很符合我的要求。


目前 Intype 最新版才 0.3 虽然功能还不是很多，但已经完全符合我的要求了，心里哪个爽呀！
