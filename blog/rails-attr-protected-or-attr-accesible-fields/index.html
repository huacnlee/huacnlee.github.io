Rails 使用参数提交表单时，要注意保护敏感字段

刚接触 Rails 的人都会对 Rails form 实际特别喜爱，因为它让我们省时省力，就算遇到有 100 多个字段的表单，也能够几下就做出来了，因为在服务端不用再去对每个字段分别写文本框与字段的赋值。

但是如果没有注意保护，使用 Model.create(params[:model]) 的方式提交会有很大的安全漏洞。

下面来看一个例子：

有用户表 [users]

  • id
  • login [用户名]
  • passwd [密码]
  • nick_name [昵称]
  • email [Email]
  • state [状态]
  • group_id [组 [1 管理员，2 编辑，3 普通用户]]
  • exp [经验值]
  • money [金币]
  • level_id [等级]
  • created_at
  • updated_at

    如何保护？

    在 Model 里面使用 attr_accessible 或 attr_protected 详见：ActiveRecord::Base 文档

    特别需要更改保护字段的时候，需要使用 @model.money = 55 这样的方式赋值，而直接 @model.update_attributes(params[:model]) 这总方式会把保护字段过滤掉。
